The Problem: Why Dropbox Search Wasn't Enough

Who I Built This For

Designers. They needed a fast way to find PSDs from past client work so they didn't have to recreate assets already completed.

Sales. They needed quick access to past work PNG examples to inspire potential clients without pulling designers away from their workflow.

The pain

At Sock Club, our design team had more than 100k+ creative assets stored across Dropbox. Filenames, metadata, and tags varied wildly, and searching for anything required either tribal knowledge or pure guesswork. Designers and the sales team weren't searching for filenames. They were searching for concepts.

And the more the library grew, the harder it became to navigate.

Even Dropbox's AI-powered search product, Dash, couldn't solve this. It was built for general file discovery, not semantic visual search across creative assets. We needed something that understood concepts, not just keywords.

Real scenarios

Four scenarios appeared constantly:

  1. Client sends over a sock design with little to no details about the past associated deal.
  2. Design wants to check if artwork / logo has already been bitmapped.
  3. Design wants to see examples of past work sent to Clients all in one place, example "Denver Airport".
  4. Sales needs to quickly find past work examples for client requests, like "Do you have any Halloween styled past work?"

Time lost

Team members were spending 15 to 30 minutes digging through nested folders for every search. Many would ask a more tenured designer if they remembered where a file lived or would eventually just recreate the asset because too much time had been lost.

This wasn't a small annoyance. It caused constant interruptions, slowed sales conversations, and quietly drained energy from the design team.

The problem wasn't storage, it was discovery.

The Solution: Semantic Search with AI Embeddings

What semantic search is

Semantic search finds items based on meaning instead of text. Embeddings convert images or text into mathematical representations that cluster similar concepts together.

Why embeddings work for creative content

Embeddings capture themes like color, composition, and mood, letting users search using concepts instead of filenames.

Building the Embedding Pipeline

Choosing the model

Vertex AI's multimodal model delivered stable, high quality embeddings for both images and text, and at the time ranked as one of the top performing embedding models on the MTEB (Massive Text Embedding Benchmark) leaderboard.

Processing 100k+ assets

The pipeline ran in batches with configurable worker concurrency, retry logic, exponential backoff, progress tracking, and logging. It also skipped any files already migrated or embedded.

Generating embeddings

Images were converted to PNG when needed, processed through Vertex AI to generate high dimensional vectors, and then upserted into Pinecone.

Setting Up the Vector Database (Pinecone)

Why Pinecone

Pinecone handles vector indexing and similarity search with almost no operational overhead.

Index design

Cosine similarity, the model's native 1408 dimensional vectors, and rich metadata for filtering.

Vector ID strategy: Design assets use Dropbox IDs for stability across S3 reorganizations, while sales assets use S3 key hashes for backward compatibility. This prevents orphaned vectors and duplicate embeddings.

Ingesting embeddings

Batch upserts kept ingestion fast and reduced network overhead.

Building the Search API (FastAPI)

Endpoint design

The API takes a text or image query, embeds it, and returns the closest matches.

Query processing

Same model, same dimensions, consistent vector space.

Similarity search

Top 100 cosine similarity results, optionally filtered by metadata.

Response formatting

Each result returns an ID, similarity score, preview URL, and asset metadata, along with a direct link into the associated HubSpot Deal and internal portal link for more context. It also includes a download button so designers receive the original PSD rather than the PNG used for embedding. Download access is protected through Cognito authorization rules to ensure only internal users can retrieve original client files.

FastAPI made it easy to keep latency low while supporting real time search.

Deployment and Infrastructure

Hosting strategy

All application code (Next.js and FastAPI) is deployed on Vercel. Asset storage lives in AWS S3, and vectors live in Pinecone.

Environment setup

Secure key storage across services, along with Cognito user pools and app client configuration to protect internal access to proprietary assets. The system also used separate environments for local development, preview deployments, and production to ensure safe testing, isolation, and reliable rollout of new features.
